8. Error Handling Best Practices 
8.1. Use Exceptions for Exceptional Situations 
• Use exceptions to handle unexpected errors, not for flow control. 
• Global Exception Handling: Implement a global exception handler using middleware to 
catch unhandled exceptions and log them appropriately.

8.2. Specific Exception Types 
• Catch and throw specific exceptions rather than generic ones for clearer error handling. 
• Example: 
try 
{ 
// Code that might throw exceptions 
} 
catch (FileNotFoundException ex) 
{ 
// Handle file not found specifically 
} 
catch (IOException ex) 
{ 
// Handle IO-related errors 
} 
8.3. Provide Meaningful Messages 
• When throwing exceptions, include clear messages describing the error context. 
• Example: 
throw new InvalidOperationException("Cannot process order because it is already completed.");

8.5. Log Exceptions 
• Log exceptions at the point they are caught, capturing relevant context information. 
• Example: 
catch (SqlException sqlEx) 
{ 
Logger.LogError(sqlEx, "Database operation failed for Order ID {OrderId}", orderId); 
throw; // Propagate the exception 
} 
8.6. Centralized Exception Handling 
• Implement centralized exception handling for consistent management of unhandled 
exceptions. 
8.7. Avoid Swallowing Exceptions 
• Never catch exceptions without handling or logging them to prevent silent failures. 
• Anti-pattern Example: 
catch (Exception) 
{ 
// Avoid this pattern 
} 
8.8. Custom Exceptions 
• Create custom exception classes when necessary to represent specific error conditions. 
• Example: 
public class OrderProcessingException : Exception 
{ 
public OrderProcessingException(string message) : base(message) { } 
}

9. Best Approach Between Using Statements and Finally Blocks for Cleanup 
Best Practice: Use using for Cleanup When Possible 
• The using statement is a syntactic shortcut that ensures the deterministic disposal of objects that 
implement IDisposable. 
• Preferred for managing unmanaged resources like file streams, database connections, etc. 
Example: 
using (var stream = new FileStream("file.txt", FileMode.Open)) 
{ 
// Use the stream 
} // stream.Dispose() is called automatically here 
Advantages: 
• Cleaner and more concise. 
• Automatically calls Dispose(), even if an exception occurs. 
• Easier to read and maintain. 
Use finally for Cleanup Only When using Isn’t Feasible 
• A finally block ensures execution regardless of exceptions. 
• Use when the resource must outlive a block or conditional disposal is needed.
